# 8. Mysql Connection

## 이 브랜치의 목표

애플리케이션을 실제 MySQL 환경과 연결하고, 실행 설정을 로컬 개발 환경에 맞추는 단계입니다.

## 여기서 배우는 것

- `application.yaml` 에서 데이터베이스 연결 정보를 관리하는 방법
- 환경변수 기반 설정의 기본 형태
- JPA 설정이 실제 DB와 연결될 때 고려할 점

## 지금 코드에서 볼 파일

- `build.gradle`
- `src/main/resources/application.yaml`

## 왜 필요한가

도메인과 API를 구현하는 것만으로는 실전과 거리가 있습니다. 실제 데이터베이스 연결을 경험해야 테이블 생성, 설정값 관리, 로컬 실행 환경 구성까지 포함한 백엔드 개발 흐름이 완성됩니다.

## 아직 아쉬운 점

- Study 아래에서 관리할 하위 도메인이 없다.
- 권한과 인증이 실제 도메인 작업에 충분히 연결되지는 않았다.
